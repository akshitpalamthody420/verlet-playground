What it does

Particles connected by constraints

Verlet Playground simulates particles using Verlet integration. Each particle stores its current and previous position, and motion is produced by comparing the two positions over time.

Particles are connected with distance constraints. When many constraints are combined, they can form ropes, cloth sheets, and soft-body shapes. The browser version below recreates the main idea of the C++ SFML project so the simulation can run on GitHub Pages.

How to use it

Controls

Input Action
Left MouseDrag nearest particle
Right MousePin or unpin nearest particle
Middle MouseCut constraints near the cursor
SpaceSpawn rope at cursor
CSpawn cloth at cursor
BSpawn soft body at cursor
WToggle wind
TToggle tearing
GToggle gravity
Delete / BackspaceClear simulation

Desktop version

Build the C++ SFML project

Requirements

  • C++17 or newer
  • CMake
  • SFML 2.5 or newer

Build

mkdir build
cd build
cmake ..
cmake --build .
./VerletPlayground

macOS with Homebrew

brew install cmake sfml@2

mkdir build
cd build
cmake -DCMAKE_PREFIX_PATH="$(brew --prefix sfml@2)" ..
cmake --build .
./VerletPlayground
